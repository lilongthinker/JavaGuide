mysql的所有变更均存储在binlog中，通过订阅binlog的变更，可以让业务只专心维护数据，其余的事情交给订阅端来处理。 其实仔细想想，只要客户不是立刻要看的信息都可以是异步的。在系统正常运转的情况下，这个时间是可以控制在1秒以内的，慢的情况也是在10秒以内。当然如果这个订阅处理链路发生问题的时候系统也要能够随着订阅延迟的问题解决后自动愈合，具有一定的自愈性。
